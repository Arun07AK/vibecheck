Architecture Overview

Use this when judges ask "explain the technical architecture" or "how does X work under the hood"

1. Static Analysis Pipeline

1
4 scanners run in Promise.all — truly parallel, not sequential. Each scans every file recursively, skipping node_modules/.git/dist
2
Secret Scanner — 20 compiled regex patterns: AWS AKIA[0-9A-Z]{16}, Stripe sk_live_, JWT eyJ..., OpenAI sk-, database URLs, private keys
3
Dependency Scanner — parses package.json, hits OSV.dev API in real-time for CVE data (GHSA IDs). Also checks against a list of 20+ known hallucinated npm packages that AI invents
4
PII Scanner — 6 patterns: email, phone, Aadhaar (India-specific), SSN, credit card, console.log(password/secret)
5
Code Smell Scanner — 10 patterns: eval(), empty catch, innerHTML XSS, SQL concatenation, hardcoded localhost, CORS wildcard, sync I/O
// All 4 scanners run simultaneously
const [secrets, deps, pii, smells] = await Promise.all([
  secretScanner.scan(repoPath),
  depScanner.scan(repoPath),
  piiScanner.scan(repoPath),
  codeSmellScanner.scan(repoPath),
]);

2. AI Simulation Engine (the USP)

MODEL CONTEXT PROTOCOL (MCP) AGENTIC LOOP
1
Start target app on a random port via spawn('node', ['server.js']) with PORT env var. HTTP poll until responsive.
2
Spawn Playwright MCP server via @modelcontextprotocol/sdk Client + StdioClientTransport. Headless Chromium browser.
3
List 21 browser tools from MCP: browser_navigate, browser_click, browser_fill, browser_snapshot, browser_evaluate, browser_console_messages, browser_handle_dialog...
4
Convert MCP tool schemas to Anthropic API tool format. Pass to Claude Sonnet 4.6 with a security-testing system prompt.
5
Agentic loop (up to 15 turns):
Claude calls browser_navigate → MCP executes → returns accessibility tree (NOT screenshots)
Claude calls browser_fill → types XSS payload into input
Claude calls browser_click → submits form
Claude calls browser_evaluate → runs JS to check DOM
Each tool result feeds back → Claude decides next action
6
Post-loop verification:
Check console for runtime errors (deduplicated, metadata filtered)
Evaluate DOM: innerHTML vs textContent comparison to detect real unescaped XSS
browser_handle_dialog detection = confirmed XSS (alert() actually fired)
// The MCP agentic loop
for (let turn = 0; turn < 15; turn++) {
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-6',
    tools: anthropicTools, // 21 MCP browser tools
    messages,
  });
  // Route tool calls to MCP server
  for (const block of response.content) {
    if (block.type === 'tool_use') {
      const result = await mcpClient.callTool({
        name: block.name, arguments: block.input
      });
    }
  }
}

Key insight: Claude doesn't see screenshots. It gets the accessibility tree — every element with role, name, and state. That's why it reliably finds inputs, buttons, and forms without guessing CSS selectors.

3. Scoring Algorithm

score = 100
CRITICAL issues: -15 each (leaked keys, eval, confirmed XSS)
HIGH issues:    -8 each  (passwords, innerHTML, PII logging)
MEDIUM issues:  -3 each  (empty catch, email exposure)
LOW issues:     -1 each  (console.log)
score = Math.max(0, score)

Grade: A+ (97+) → F (<40)
Verdict: GO (70+) | WARNING (40-69) | NO-GO (<40)

4. Knowledge Base

1
SQLite with WAL mode for concurrent reads. Three tables: scans, issues, patterns
2
Every scan does: INSERT INTO patterns ... ON CONFLICT(error_type) DO UPDATE SET frequency = frequency + 1
3
After 34 scans: 50 unique patterns, 11,540 total issues. Top: Email leaks 4,335x, Console.log 1,196x

5. Five AI Domain Agents

1
All 5 run in Promise.all against Claude Sonnet 4.6 — parallel, not sequential
2
Each gets: file list + scan results summary. Each has a domain-specific system prompt.
3
Each outputs: assessment + copy-paste fix prompt the developer gives to Cursor/Copilot
4
Domains: Security, UX, Performance, Scalability, Production Readiness

6. GitHub Action CI/CD

1
String template generates .github/workflows/vibecheck.yml
2
Clones VibeCheck repo on GitHub's Ubuntu runners, runs 4 scanners against user's code
3
Score below 40 → process.exit(1) → CI fails. Tested live on real repo — push failed.

If asked: "What was the hardest technical challenge?"

Arun says: Connecting Claude to the browser via MCP. The Anthropic API returns tool_use blocks, but we need to route them to Playwright's MCP server and feed the results back. We built a bridge: MCP tool schemas get converted to Anthropic tool format, tool calls get routed to the MCP client, and tool results get fed back as tool_result messages. It's a custom agentic loop — 15 turns of Claude driving a real browser through structured tool calls, not screenshots.
